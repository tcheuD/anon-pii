The case
Four points

Why not just sed it out.

Because a black marker is permanent, and four of these are things a regex in your shell history will not give you.

01

One owned roundtrip

anon-pii run sends tokens to one child process and restores known tokens from stdout. That invocation's mapping stays in memory.

02

One native CLI

The default path needs no Python environment, Docker container, long-running service, or ML runtime.

03

Validation where formats allow it

Checksum-backed identifiers use validators such as Luhn and IBAN mod-97; ambiguous patterns use context. Neither removes false positives or false negatives.

04

A versioned quality contract

Synthetic positive, negative, normalization and structured-format cases run in CI. Claims stay tied to a corpus and revision instead of a blanket accuracy ranking.

Install.

A local data-minimization aid, not a compliance guarantee — read the security model before trusting it with anything real.

Pre-built binary — linux x86_64, macos x86_64 / aarch64
curl -L https://github.com/tcheuD/anon-pii/releases/latest/download/anon-pii-macos-aarch64.tar.gz | tar xz
sudo mv anon-pii /usr/local/bin/
From source
git clone https://github.com/tcheuD/anon-pii && cd anon-pii
cargo install --path .
