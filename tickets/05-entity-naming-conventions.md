# Ticket #05: Align Entity Naming Conventions

**Priority:** High
**Complexity:** Medium
**Status:** DONE
**File:** `src/main.rs`

## Description

Python uses Presidio-style long entity names. Rust uses short names. This creates incompatibility: mappings generated by one implementation cannot be restored by the other.

## Options

### Option A: Adopt Python/Presidio naming (recommended)

Change Rust entity types to match Python:
- `EMAIL` → `EMAIL_ADDRESS`
- `FR_PHONE` → `FR_PHONE_NUMBER`
- `AIRCRAFT` → `AIRCRAFT_REGISTRATION`
- `FLIGHT` → `FLIGHT_NUMBER`
- `IP` → `IP_ADDRESS`

**Pros:** Cross-implementation mapping compatibility, industry-standard naming.
**Cons:** Breaking change for existing Rust users, longer token strings.

### Option B: Adopt Rust short naming

Change Python entity types to match Rust. Requires changes in the Python codebase.

**Pros:** Shorter tokens, less visual noise.
**Cons:** Breaks Presidio compatibility, non-standard.

### Option C: Support both with aliases

Accept both naming styles during restore. Use one canonical form for generation.

**Pros:** Maximum compatibility.
**Cons:** Complexity.

## Implementation (Option A)

Update `entity_type` field in all `PiiPattern` entries:

```rust
PiiPattern { name: "email", entity_type: "EMAIL_ADDRESS", ... },
PiiPattern { name: "fr_phone_intl", entity_type: "FR_PHONE_NUMBER", ... },
PiiPattern { name: "aircraft_fr", entity_type: "AIRCRAFT_REGISTRATION", ... },
PiiPattern { name: "flight_amelia", entity_type: "FLIGHT_NUMBER", ... },
PiiPattern { name: "ipv4", entity_type: "IP_ADDRESS", ... },
```

Update ALL tests that reference the old names.

## Breaking Change Warning

This changes the output format. Existing mapping files generated by Rust will not be compatible with the new naming. Consider:
- Bumping version to 0.2.0
- Documenting the change in CHANGELOG
